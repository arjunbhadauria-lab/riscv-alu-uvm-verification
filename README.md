# RISC-V RV32I ALU: UVM Verification with AI-Assisted Test Generation

A complete UVM verification environment for a RISC-V RV32I Arithmetic Logic Unit, featuring industry-standard constrained random verification, directed testing, self-checking scoreboards, functional coverage collection, and an AI-assisted test generation experiment comparing LLM-generated stimulus against constrained random approaches.

## Results Summary

**UVM Testbench (EDA Playground, Aldec Riviera-Pro):**
- 2031 transactions, **0 failures**
- **83.33% functional coverage**
- Zero UVM_ERROR, zero UVM_FATAL

**AI vs Random Test Generation (Python):**
- AI-generated tests achieved **63.6% average cross coverage** with 56 tests
- Constrained random achieved **37.9% average cross coverage** with 56 tests
- AI tests reached coverage goals **25.7 percentage points** more efficiently

## Design Under Test

The ALU implements all ten RV32I base integer operations:

| Operation | Encoding | Description |
|-----------|----------|-------------|
| ADD       | 0000     | Addition |
| SUB       | 0001     | Subtraction |
| AND       | 0010     | Bitwise AND |
| OR        | 0011     | Bitwise OR |
| XOR       | 0100     | Bitwise XOR |
| SLL       | 0101     | Shift Left Logical |
| SRL       | 0110     | Shift Right Logical |
| SRA       | 0111     | Shift Right Arithmetic |
| SLT       | 1000     | Set Less Than (signed) |
| SLTU      | 1001     | Set Less Than (unsigned) |

The ALU also generates four status flags: zero, carry, overflow (signed), and negative.

### Components

- **Sequence Item**: Randomisable transaction with weighted distribution constraints targeting boundary values (0x0, 0xFFFFFFFF, 0x7FFFFFFF, 0x80000000)
- **Driver**: Drives constrained random and directed stimulus onto the DUT interface
- **Monitor**: Passively observes interface signals and broadcasts transactions via analysis port
- **Scoreboard**: Contains a reference model that independently computes expected results and checks all outputs and flags against actual DUT behaviour
- **Coverage Collector**: Tracks functional coverage across operations, operand ranges, flag outputs, and cross coverage between operations and operand categories

### Test Strategies

**Directed Test**: Hand-crafted corner cases including:
- All operations with zero operands
- Signed overflow (0x7FFFFFFF + 1) and underflow (0x80000000 - 1)
- All operations with all-ones operands
- Shift boundary tests (shift by 0 and shift by 31)
- SLT vs SLTU difference with signed negative values

**Random Test**: 1000 constrained random transactions with weighted distribution favouring boundary values

**Full Test**: Directed corner cases followed by 2000 random transactions for maximum coverage

## AI-Assisted Test Generation

The `scripts/ai_test_gen.py` script explores using LLM-generated test vectors as an alternative to constrained random stimulus. Test vectors were generated by prompting Claude (Anthropic) with the ALU specification and requesting coverage-maximising test cases. These are compared against constrained random tests matching the UVM testbench distribution.

### How It Works

1. **AI test vectors**: 56 targeted tests generated by an LLM, each with a specific verification intent (overflow detection, boundary shifts, signed vs unsigned comparison edge cases)
2. **Random test vectors**: 56 constrained random tests using the same weighted distribution as the UVM testbench
3. **Coverage comparison**: Both sets are run through an identical Python reference model with coverage tracking that mirrors the UVM covergroup
4. **SystemVerilog export**: AI tests are exported as UVM sequence items that can be directly included in the testbench

### Key Findings

- AI tests achieved **25.7% higher cross coverage** by systematically targeting every ALU operation with boundary value operands
- Random tests still provide value through broad design space exploration, with both approaches achieving 100% operation and flag coverage
- The ideal verification strategy combines both: AI-targeted corner cases for efficient coverage closure, followed by constrained random stimulus for breadth, which is exactly what the `alu_full_test` implements

## Running the Simulation

### UVM Testbench (EDA Playground)

1. Go to [edaplayground.com](https://www.edaplayground.com)
2. Paste `rtl/rv32i_alu.sv` and `tb/alu_interface.sv` in the right (design) panel
3. Paste the combined testbench file in the left (testbench) panel
4. Select **Aldec Riviera-Pro** as the simulator
5. Tick **UVM/OVM** and select **UVM 1.2**
6. Add `+UVM_TESTNAME=alu_full_test` in the run options
7. Click **Run**

### AI Test Generation (Python)

```bash
cd scripts
python ai_test_gen.py
```

For live LLM generation (requires a Gemini API key):
```bash
python -m pip install google-generativeai
export GEMINI_API_KEY="your-key"    # Linux/Mac
$env:GEMINI_API_KEY="your-key"      # Windows PowerShell
python ai_test_gen.py --live
```
